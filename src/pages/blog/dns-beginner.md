## 1. DNSの基本概念

### ゾーン (Zone)
- ドメインに対応する **名前空間の管理単位**。
- そのゾーンに含まれるレコードの集合を管理する。
- 管理責任者は **SOAレコード**で定義。

### 権威 (Authoritative)
- ゾーン情報の **正本（オリジナル）を保持して回答するサーバ**。
- キャッシュサーバ（非権威応答）と区別される。

---

## 2. 主なDNSレコードとセキュリティ上の効果

| レコード種別 | 用途 | セキュリティ上の効果・リスク |
|--------------|------|------------------------------|
| **A / AAAA** | ホスト名 → IP | IPなりすまし・キャッシュ汚染の標的 |
| **MX** | メール配送先サーバ | 偽装でメール盗聴・不達攻撃が可能 |
| **CNAME** | 別名定義 | サブドメインテイクオーバーのリスク |
| **PTR** | IP → ホスト名 (逆引き) | 不整合があるとスパム判定要因 |
| **TXT (SPF)** | メール送信許可IP宣言 | スプーフィング防止（MAIL FROM） |
| **TXT (DKIM)** | 電子署名検証鍵 | メール改ざん検知 |
| **TXT (DMARC)** | SPF/DKIMポリシー宣言 | ユーザ表示アドレスの整合性保証 |
| **CAA** | 証明書発行許可CA指定 | 誤発行・不正発行を防止 |
| **DNSKEY / RRSIG** | DNSSECの鍵/署名 | 応答改ざん防止 |

---

## 3. 再帰問い合わせとキャッシュポイズニング

### 再帰問い合わせの流れ
1. クライアント → リゾルバ（キャッシュサーバ）に問い合わせ  
2. リゾルバが権威サーバへ段階的に問い合わせ（root → TLD → 権威）  
3. 応答をキャッシュし、クライアントに返答  

### キャッシュポイズニング
- リゾルバが権威サーバへ問い合わせ中に、**偽の応答を外部から送りつけてキャッシュを汚染**する攻撃。
- 主に標的となるのは **Aレコード（正規サイトのIP偽装）**。

---

## 4. カミンスキー攻撃と従来型キャッシュポイズニング

| 項目 | 従来型 | カミンスキー攻撃 |
|------|---------|------------------|
| アプローチ | 偶発的にIDやポートを当てる | 大量のサブドメインを生成し競合を増幅 |
| 成功率 | 低い | 高い（高速で大量試行可能） |
| 対策 | - ソースポートランダム化<br>- トランザクションIDランダム化 | - 上記に加え **DNSSEC** |

---

## 5. UDPとTCPの違い（偽装容易性）

| 項目 | UDP | TCP |
|------|-----|-----|
| 接続性 | コネクションレス | コネクション型 |
| 検証要素 | 16bitトランザクションID + 送信元IP/ポート | 3-way handshakeによりシーケンス番号同期必要 |
| 偽装難易度 | 容易（小さいID空間を推測可能） | 難しい（正しいシーケンス番号同期が困難） |

---

## 6. DNSSECによる改ざん防止

- 各レコードに **署名 (RRSIG)** を付与し、**DNSKEY** により検証可能。
- 応答が権威サーバから発行された真正なものか検証できる。
- 攻撃者が偽応答を注入しても、署名検証で検出される。

---

## 7. メール関連セキュリティ（SPF）

### SPFの仕組み
- ドメインのDNSに「このドメインからメールを送信可能なIP」を宣言。
- 受信側MTAは送信元IPと比較し、許可外なら拒否や隔離。

### SPFレコード例

```txt
example.com. IN TXT "v=spf1 ip4:203.0.113.0/24 include:_spf.service.net -all
```

### SPFの効果と限界
- ✅ **MAIL FROM のなりすまし防止**  
- ❌ **ヘッダFrom偽装は防げない** → DMARCと併用必須  
- ❌ **転送メールでSPF失敗する問題** → SRSやDKIMで補完  

---

## 8. 代表的対策まとめ

| 攻撃 | 対策 |
|------|------|
| キャッシュポイズニング | ソースポートランダム化・IDランダム化・DNSSEC導入 |
| カミンスキー攻撃 | 同上 + サブドメイン応答の厳格化 |
| メールスプーフィング | SPF, DKIM, DMARC の三点セット |
| 証明書誤発行 | CAAレコードの設定 |
| サブドメインテイクオーバー | CNAMEの利用監査・外部サービス管理 |
