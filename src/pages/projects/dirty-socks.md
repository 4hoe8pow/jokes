## 1. 概要
**タイトル**：Dirty Socks
**キャッチコピー**：汚れるほど、強くなる。

**コンセプト**：
- 靴下たちが汚れるほど強くなるが、UIが汚れて見えにくくなる
- 毎回異なるローグライクマップ探索＋ハクスラ育成体験
- 隠しヒントや裏コマンドで遊びと発見を演出
- ダーティネスの蓄積と戦略性のジレンマがコア体験

**対象プラットフォーム**：iOS限定
**プレイ形式**：シングルプレイ、ステージ5〜10分程度

---

## 2. 世界観
- 片方だけよく消える靴下たちが主人公
- 悪友のようなキャラクターで、正義感ゼロ
- UI（テキストログ＋ネットワークマップ）が探索の目
- 汚れや演出で「冒険している感」を演出

---

## 3. ノード（場所）の種類
| ノード                 | 説明                                      |
| ---------------------- | ----------------------------------------- |
| 敵                     | 戦闘発生。倒すとHATCHのヒント取得         |
| 洗浄                   | 汚れを軽減。戦闘力や効果はやや低下        |
| ワープ                 | ペアで移動可能な転送ポイント              |
| スキット               | 寸劇・ミニドラマ・ホラー。HATCHヒントあり |
| 大輪軒                 | 回復場所。ヒントも少し取得可能            |
| START                  | ゲーム開始地点                            |
| アンロックハッチノード | キャラを配置すると次ステージ進行可        |
| ハッチノード           | 偽装・非表示。発見で脱出成功              |

- **期間限定イベント**：特殊スキットノードや限定大輪軒ノードを出現させることが可能

---

## 4. 基本の遊び方
- ハニカムパネル風マップを探索
- Socksキャラクター4体を操作
- アンロックハッチに1体配置＋別の1体でHATCH到達が勝利条件
  - 靴下同士がすれ違えないパズル(単線)制約
- 全滅でステージ失敗
- ノード通過で**ダーティネス**蓄積
  - **軽度**：文字ログの濁り
  - **中度**：モザイク・グリッチ演出入り
  - **重度**：UIがほぼ機能せず視覚的制約
- 洗浄ノードでダーティネス軽減可能
- ダーティネスは戦闘・探索を有利にするバフ効果
- 期間限定イベントノードで限定報酬を提供可能

---

## 5. キャラクターと育成
- Socksキャラクターは8つの個性パラメータを持つ
- XPを自由に振り分け可能
- ダーティネス蓄積や戦闘結果に影響し、探索や戦略に深み

| パラメータ    | 意味       | ダーティネスとの関係               |
| ------------- | ---------- | ---------------------------------- |
| Dolour        | 嘆き深さ   | ダーティネス蓄積量に影響           |
| Baseness      | 浅ましさ   | 戦闘・トラップ回避に影響           |
| Vexation      | もどかしさ | 移動コスト・選択肢に影響           |
| Ineptness     | 拙さ       | 戦闘失敗率に影響                   |
| Precarity     | 危うさ     | UI汚染による探索制約に影響         |
| Karmic Burden | 業の深さ   | 特殊スキットやイベント発生率に影響 |
| Fragility     | 脆さ       | 全滅リスクや回避判定に影響         |
| Indecorum     | はしたなさ | NPC反応・隠しヒント出現率に影響    |

### 長期的エンゲージメント
- **ステータス持越し**による継続的な成長感の提供
- **持久戦ベースの進行設計**
  - 成長曲線は「指数関数的」ではなく「対数関数的」に設計
  - プレイ初日：到達可能現場数 *約3*
  - 継続30日：到達可能現場数 *約6*
  - 継続1年：到達可能現場数 *約12*

## 6. 報酬
- **エメラルド**：通貨。スキン購入や広告削除など
- **XP**：キャラクター育成に使用
- **入手方法**：
  - ステージクリア報酬
  - 日次ミッション
  - 敵図鑑コンプリート（図鑑登録・発見時に報酬）
  - 裏示唆・隠しノード発見報酬

---

## 7. 敵図鑑
- 遭遇した敵を登録し、図鑑完成度で報酬
- 種類やレア度、過去の遭遇履歴を記録
- 図鑑完成でエメラルドやXPを獲得

---

## 8. HATCH探索と示唆
- 示唆は**確定情報**：方角やノード種類、グループ範囲が分かる
- 特殊演出で探索のワクワク感：
  - 文字逆回転、ブラックアウト、次回予告、金文字・ゼブラ柄、リールロック
- テキストログUIのみ、太字で重要情報を強調
- 超低確率イベント(1/65536)として示唆ではなく希少靴下との契約

---

## 9. 裏コマンド・隠し示唆
- 確定情報ではなく「ほのめかし」として演出
- 発見体験はプレイヤー驚き＋SNS拡散狙い
- **制約**：
  - 1プレイ1回まで、または特定状況下でのみ発動
- トリガー例：
  - 画面端タップ3回 → HATCH方向が一瞬点滅
  - デバイスシェイク → 「遠くから風の音」ログ＋端末バイブ
  - START地点で特定順スワイプ → 「靴下の古い記憶」演出＋追加ヒント
  - 15秒無操作 → UIにじみ、HATCH方向を暗示
- パチスロインスパイア：
  - リールロック、文字逆回転、ブラックアウト、金文字・ゼブラ柄、次回予告演出など

---

## 10. ステージ生成
- 無向グラフ生成によりローグライクマップを自動生成
- **理論的手法**：
  - 細胞自動機（Cellular Automata）でノード密度・分布
  - 力学的レイアウトで見やすく配置
  - BSPで連結性と通路バリエーションを確保
- **ノードタイプ配置**：
  - 敵・洗浄・スキット・大輪軒・アンロックハッチ・HATCHをランダム化
  - 期間限定イベントノードもシード指定で配置可能
- **探索性保証**：
  - START～HATCH間の連結必須
  - Union-Findで連結性検証
  - Dijkstra / A*で最短経路評価、TSPライクで戦略ルート検討
- エッジ重み = 移動コスト、ダーティネス蓄積・回復を考慮
- 無限生成可能で毎回異なる探索・戦略・報酬構造を提供
