## 概要
**Dirty Socks** は、汚れた靴下たちを各「現場」から脱出させるデータドリブン・ローグライクゲームです。グラフィカルな地図・視覚表示は意図的に存在しません。すべて数値・統計指標といった定量的情報から状況を推測し、間接的に介入します。

## ゲーム目的
靴下4体を誘導し、1体でも脱出ハッチにたどり着くとその現場はクリアとなり、次の現場 STAGE2 がスタートされます。4枚が力果てるとゲーム終了です。可能な限り多くの現場で脱出させ、**累積スコア（ゴルフ方式のPAR差）** を稼ぎましょう。現場ごとに動ける距離は決まっており、なるべく少ない移動距離で脱出することがロースコアへ繋がります。

## コアルール
- 各現場 = スタート（投入地点） + ゴール（複数の脱出ハッチ） + イベントノード
- 現場1を脱出すると次の現場2へ。靴下のステータスは継続（回復なし）。  
- プレイヤーは靴下に直接指示できない。コックピットで **環境変数** を調整して間接介入。
- ダーティネスが溜まると靴下は強くなり、敵を倒しやすくなるが、見える情報が濁る。 
- 運要素演出(RNG）は存在するが、**進行結果を左右しない**スパイス程度。

## 確認できる主な指標
- 現在のゴールまでの距離(最重要指標)  
- 各靴下のダーティネス（汚れ度）・HP・到達確率（推定）  
- 各イベントに関わる指標：温度、湿度、など１０種程度のパラメータ群
> 備考：ダーティネスが高いほど靴下は強くなるが、コックピットの表示は劣化（部分的マスク、ノイズ）します。

## 追手やギミック(敵対勢力)
- 洗濯ばさみ、乾燥機、掃除機、洗濯かご、柔軟剤の香り 等

## 操作体験の設計（プレイヤーができること）
1. Socksたちは自律的に現場内を移動するため、プレイヤーは先回りして無駄な通路を封鎖し、効率的な進路を確保できる。
2. 勝てなさそうな敵が現れる前に、落とし穴などを設置して撃退することも可能。ただし、回収を忘れるとSocksも落下するリスクがある。
3. ステータス強化や脱出支援用の装置を各ノードに設置し、Socksたちの進行をサポートできる。

わらわら歩くSocksたちを、装置や罠で戦略的に狙った場所へ運ぶ感覚は、タワーディフェンスとカーリングが融合したような体験になる。

## UXガイドライン

| 課題として感じること | カテゴリ | ゲームでの工夫（施策） |
|---|---|---|
| 重要な情報が多すぎて何を見ればいいかわからない | 情報可読性 | 主要な指標は常に見やすく表示。ダーティネス上昇時は情報の一部が少し見えにくくなるが、ノイズパターンで敵や隠しイベントのヒントを示す。必要に応じて情報をフィルタ可能。 |
| ノイズや情報の一部が意味のないランダムに見える | 情報可読性 | ノイズには意味があり、追手の接近やイベントの発生を示すサインとして活用。読み解く楽しみを提供。 |
| 自分の操作が靴下にどのように影響しているか分からない | 技術介入 | 環境変数や装置を変更すると、HP・ダーティネス・到達確率の変化をすぐに数値とメッセージで確認できる。 |
| 操作の結果が後で振り返れず、次の作戦に活かせない | 技術介入 | 行動ログや介入前後の指標差分を確認可能。自分の判断がどう影響したか分析できる。 |
| 画面が情報でごちゃごちゃしている | UI/UXデザイン | 重要な情報は常に目立つように配置。数値の変化は色や点滅で直感的にわかるようにし、操作も簡単で直感的。 |
| 危険や状態変化がよくわからない | UI/UXデザイン | ダーティネスや危険時にフォントや色で状態を表示し、エラーや警告も分かりやすく伝える。 |
| 靴下が数字だけで味気なく感じる | 世界観 | 各靴下に名前や性格、短い紹介文を付け、個性や物語を感じられるようにしている。 |
| 現場やイベントの状況がイメージできない | 世界観 | ステージ開始時に現場の特徴や危険をテキストで説明。イベント時も短文ログで状況を伝える。 |
| ダーティネスが単なるペナルティに感じる | ゲームシステム | ダーティネスが高いと攻撃力が上がったり、隠しイベントや危険区域通過が可能になるなど、戦略的な要素として活用できる。 |
| どの道を選ぶと有利か分かりにくい | ゲームシステム | 到達確率や平均移動距離、次のイベント遭遇確率などをリアルタイムで表示し、戦略の参考にできる。 |
| どの環境変数が影響しているかわからない | ゲームシステム | 温度や湿度などの環境変数が靴下や追手に与える影響を数値やグラフで示し、調整の参考にできる。 |
| 運要素で計画が台無しになると感じる | 運要素 | RNGは結果に大きく影響せず、巡回ルートの微調整やイベントタイミングなどの「小さな変化」に限定。 |
| 運の影響が理解できずフラストレーションになる | 運要素 | RNGの発生確率を明示し、ログ化。装置や環境調整で運の影響をやや緩和できる。 |
