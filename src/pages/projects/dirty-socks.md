## 概要
- **アプリ名**: DIRTY SOCKS  
  iOS / SwiftUI / SwiftData ベース
- **開発方針**: SwiftUI単独、UIKit/Unity/UE5 不使用  
  古典的テキストRPG風ログゲーム。やりこみ要素重視。

## アーキテクチャ
- **Composable Architecture (MVVM禁止)**  
  - View は軽量
  - `@StateObject` や `@ObservedObject` で GameState / PlayerState / SockState / EnemyState を購読
  - Action は State 側で処理

## 1. インゲーム・メカニクス基本コンセプト

**「清潔という名の恐怖からの逃走劇」**

プレイヤーは、4体の汚れた靴下（パーティ）を観測し、指令を出す「司令官」として、彼らを待ち受ける「洗濯」という名の運命から逃すことを目指す。この世界における敵やトラップは、すべて「清潔」「純粋」「無垢」な存在として描かれる。靴下たちは**汚れるほどに個性を強め、本来の力を発揮する**が、洗い清められる（デバフを受ける）と弱体化してしまう。ホラーとは、純白の恐怖である。

プレイヤーが見るUIは、靴下たちの状況を分析する**「コックピット」**そのもの。しかし、靴下の汚れは、客観的であるはずの計器盤すら「汚染」し、プレイヤーの正常な判断を蝕んでいく。

## 2. コア体験：コラプティング・コックピット (Corrupting Cockpit)

### 2.1. コアメカニクス：ダーティネス（汚れ度）
- 各靴下が持つ0%〜100%のパラメータ。セッション開始時は常に0%（真っ白）。
- 現場内のギミックやイベントを通じて変動する。
- **メリット:** ダーティネスが高いほど、靴下の基礎パラメータ（attack, durabilityなど）にプラス補正がかかり、特定のスキルが解放・強化される。**魂が強まる**状態。
- **デメリット:** ダーティネスが一定値を超えると、後述するUI/UXを通じてプレイヤーが得られる情報が欠落・汚染されていく。**理性が壊れる**状態。

### 2.2. UI/UXの二層構造：主観的ログと客観的計器
アセットの不在を「それ自体が美学」へと転換するため、UIは二層構造でプレイヤーの体験を設計する。

- **主観レイヤー（上層）：** 靴下たちの「魂の叫び」。定性的・直感的な情報。
- **客観レイヤー（下層）：** 司令官の「理性」。定量的・分析的な情報。

ダーティネスの上昇は、この二層の情報を相反する方向へ変質させる。ログはより雄弁になるが、計器はより不正確になるというジレンマが、本作の戦略性と恐怖の核心となる。

### 2.3. 主観レイヤー：朽ちるログスクリーン
- 数値パラメータを直接表示せず、ログの吹き出しのビジュアルでダーティネスを表現する。
- **0-10%:** 透明に近い清潔な吹き出し。ログの存在感が薄い。
- **20-40%:** 少し黄ばみやシミが現れ、生活感が滲み出す。
- **50-70%:** 黒ずみや毛玉風テクスチャが混ざり、文字がかすれ始める。「育っている」ことが直感的にわかる。
- **80-90%:** 枠が崩れ、文字が滲む。異様な圧を放つ。
- **100%:** 吹き出しは破片と化し、文字はノイズ混じり（例:「■■が■■した！」）となり、ほぼ判読不能になる。最強の状態であると同時に、最も情報が得にくい状態となる。

### 2.4. 客観レイヤー：乱れるインストゥルメントパネル
SwiftUIの `Gauge`, `ProgressView`, `Canvas` 等を駆使し、Glassmorphismエフェクトのかかった計器盤を構築する。

| 計器名称 | 役割 | SwiftUI活用案 |
|:---|:---|:---|
| **SYSTEM INTEGRITY** | **[マスターゲージ]** パーティの平均ダーティネスに反比例して低下。**全ての計器の乱調の元凶。** | `Gauge` (中央配置), `overlay` (亀裂エフェクト) |
| **PURITY MONITOR** | エリアの「清潔度」。高いほど危険。 | `Gauge` (針の振動、警告灯) |
| **THREAT RADAR** | キーパーやギミックの存在を示唆するソナー。 | `Canvas` + `TimelineView` (スキャンと幻影ブリップ) |
| **ACOUSTIC SIGNATURE** | 周囲の物音レベルの波形。 | `Canvas` (リアルタイム波形ビジュアライザ) |
| **VITALS MONITOR** | パーティ4体のHP。心電図のようなライン。 | `ProgressView` or `Canvas` (ラインの乱れ、[ERROR]表示) |
| **EVENT TIMERS** | 状況ログ生成やエンカウンターまでの秒読み。 | 円形 `ProgressView` (複数配置、色相変化) |

### 2.5. 計器の実装基準

| 計器                        | 実装コンポーネント案               | 設計品質・評価観点                                                                                                                                              |
| ------------------------- | ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ランプ (Indicator Light)     | GlowingOrb.swift         | RadialGradient、blur、shadowを多層的に組み合わせ、ガラス質感を高忠実度で表現。光量は外部パラメータと連動し、状態変化に応じて明滅可能。描画は `.drawingGroup()` により統合し、アニメーション時のパフォーマンスを確保。色やサイズは依存注入可能とし、汎用性を担保。 |
| ゲージ (Gauge)               | InstrumentGauge.swift    | カスタム針・目盛り描画を行い、金属光沢や反射表現を実現。SYSTEM INTEGRITY 低下時には針の振動・グリッチ・反応遅延を段階的に再現。目盛範囲や閾値を外部から設定可能とし、複数用途で利用可能。                                                  |
| レーダー (Radar Display)      | ThreatRadarView\.swift   | TimelineViewとCanvasで回転スキャンを実装し、残光効果を付与。脅威ブリップは光量・サイズで危険度を表現。SYSTEM INTEGRITY 低下時はスキャン速度変動、幻影ブリップ出現、CRTノイズ重畳を導入。描画負荷はブリップ数制御で抑制。                        |
| 波形モニター (Waveform Monitor) | WaveformMonitor.swift    | Pathベースで波形をリアルタイム描画し、自己発光感をshadowで強調。SYSTEM INTEGRITY 低下時にスパイクノイズ、信号ロスト、ブロックノイズを再現。外部データを注入可能とし、HPや音響など多用途に展開可能。                                       |
| 円形タイマー (Countdown Timer)  | CircularTimer.swift      | Shape/Animatableで円形プログレスを構築。残余時間に応じて色相を段階的に変化させ、終盤はパルスアニメーションで警告。SYSTEM INTEGRITY 低下時には進行停止・逆行・グリッチテキストを発生。アニメーション負荷は最小限化。                             |
| ログ表示域 (Log Display)       | CorruptingLogView\.swift | 各ログ行をカスタムビュー化し、ダーティネス値に応じてテクスチャ重畳・シェーダ歪曲・文字崩壊を段階的に適用。判読可能状態から崩壊状態への遷移をリアルタイム反映。LazyVStackとScrollViewReaderでパフォーマンスを最適化し、表示範囲外のエフェクト処理を抑制。              |

#### 実装例

```swift
import SwiftUI

struct GlowingOrb: View {
    let color: Color
    var size: CGFloat = 22
    var isOn: Bool = true
    // 光の強さ（0..1）
    var intensity: Double = 1.0
    var accessibilityLabel: String? = nil

    var body: some View {
        ZStack {
            // outer glow (big, very blurred)
            if isOn {
                Circle()
                    .fill(color.opacity(0.20 * intensity))
                    .frame(width: size * 2.2, height: size * 2.2)
                    .blur(radius: max(6, size * 0.35))
                    .compositingGroup()
                    .shadow(color: color.opacity(0.14 * intensity), radius: size * 0.9)
            }

            // mid glow (softer)
            if isOn {
                Circle()
                    .fill(color.opacity(0.30 * intensity))
                    .frame(width: size * 1.4, height: size * 1.4)
                    .blur(radius: max(3, size * 0.18))
            }

            // core with subtle radial highlight
            Circle()
                .fill(
                    RadialGradient(gradient: Gradient(colors: [
                        (isOn ? color.opacity(0.95) : Color.gray.opacity(0.5)),
                        (isOn ? color.opacity(0.60) : Color.gray.opacity(0.35))
                    ]), center: .center, startRadius: 0, endRadius: size)
                )
                .frame(width: size, height: size)
                .overlay(
                    Circle()
                        .stroke(Color.white.opacity(isOn ? 0.12 : 0.06), lineWidth: 0.8)
                )
                // small crisp highlight to sell 'shiny'
                .overlay(
                    Circle()
                        .fill(Color.white.opacity(isOn ? 0.08 : 0.0))
                        .frame(width: size * 0.35, height: size * 0.35)
                        .offset(x: -size * 0.18, y: -size * 0.18)
                        .blendMode(.screen)
                )
                .shadow(color: isOn ? color.opacity(0.18 * intensity) : Color.clear,
                        radius: isOn ? size * 0.6 : 0, x: 0, y: 0)

        }
        .accessibilityElement()
        .accessibilityLabel(accessibilityLabel ?? (isOn ? "Indicator on" : "Indicator off"))
        .accessibilityAddTraits(isOn ? .isSelected : [])
        // パフォーマンス：グループ化して1つのレイヤに落とす
        .drawingGroup()
    }
}
```

## 3. ゲームフロー

### 3.1. 基本サイクル：エリアごとの「静」と「動」のサイクル

ゲームはエリアごとに**「平穏フェーズ」**と**「警戒フェーズ」**を繰り返す。  
平穏では準備やリスクテイクを行い、警戒では発生したトラブルに対応する。この緩急のリズムが緊張感と達成感を生む。

---

### 3.2. フェーズ1：平穏 - 探索・準備・リスクテイク

#### エリア特性の把握
- 各エリアには明確なテーマ（例：洗濯槽ドラム、ベランダ物干し竿回廊）を設定。
- エリア名や初期ログから、起こりうるリスクや戦略のヒントを把握。

#### 状況ログ生成
- 4秒ごとに環境や靴下の状態を描写。
- 靴下パラメータによってログ文面が変化し、隠されたヒントや危険の予兆を提示。

#### 能動アクション：ダーティ・ダイブ
- プレイヤーは任意で「ダイブ」コマンドを実行可能。
- **リワード:** ダーティネス上昇、アイテム発見、ショートカット開通。
- **リスク:** HP減少、物音による警戒フェーズ移行。

#### イベント発生とAsk
- 時間経過やギミック接触でイベント発生。
- 「助ける／見捨てる」などの状況拘束型Askを提示し、ドラマ性の高い選択を強いる。

#### アクション解決：コンボ・ログ・システム
- 4体の靴下への非同期入力が順序に従って解決。
- **連携ログ:** 入力順と組み合わせによる正の連鎖反応で汚染ゲージ上昇。
- **事故ログ:** 予期せぬ組み合わせや失敗で状況悪化、警戒フェーズ移行トリガー。

---

### 3.3. フェーズ移行：平穏→警戒

**警戒フェーズ移行トリガー**
- 事故ログ発生
- ギミック解除失敗
- ダーティ・ダイブによる物音
- エリアへの長時間滞在
- キーパーによる発見（即座にチェイスへ）

---

### 3.4. フェーズ2：警戒 - 脱出・サバイバル・クライマックス

#### モードチェンジ
- コックピットUIがレッドアラートモードに移行。警告灯点滅、計器盤乱調。
- BGMや環境音が緊迫。
- 状況ログ更新間隔が短縮、キーパー遭遇率上昇。

#### チェイス・シーケンス
- 発見後、リアルタイム逃走開始。
- 状況ログ短周期更新で瞬時判断を要求。

#### 特殊アクション：サクリファイス・チョイス
- 専用コマンドとして環境利用や犠牲を選択可能。
- パーティの生存数や脱出優先度に影響。

#### 決定打：汚染アーツ
- 平穏フェーズで溜めた汚染ゲージ最大時に発動可能。

---

### 3.4. ギミック＆イベント設計

#### 設計思想
- 個別の事象をマスターで管理するのではなく、**「効果（Effects）」を組み合わせる**ことで、多様なギミックとイベントを柔軟に生成する。
- すべてのギミック/イベントは、HPや状態異常だけでなく、**必ずダーティネス（汚れ度）に影響を与える**。
- **解釈ログ:** 同じ事象でも、パーティ内の靴下のパラメータ（`logic`, `empathy`等）によってログの文体や情報量が変化する。プレイヤーはログを能動的に「解釈」する必要がある。
- **予兆ログ:** 特定の行動がフラグとなり、すぐには結果が現れず、後のログに伏線として影響を与える。過去のログが未来の恐怖を演出する。
