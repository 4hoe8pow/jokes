## 概要
**Dirty Socks** は、汚れた Socks たちを各「現場」から脱出させるスライドパズル型ローグライクです。
プレイヤーは Socks を操作してグラフ状の現場を探索し、脱出口であるハッチ到達を目指します。

特徴：
- **完全自動生成の現場**：毎回異なる構造で新鮮な探索が可能
- **心理的探索体験**：HATCHは偽装表示され、各中間ノード到達時の演出が場の状況を示唆
- **希少イベント・報酬**：低確率で発生し、探索に刺激と恩恵を提供
- **ステータス持越しによる持久戦**：現場間でキャラクター状態を引き継ぎ、長期的な成長とリプレイ性を確保
- **語りたくなる瞬間の演出設計**：自然発生と意図的演出のバランスで、SNSや実況で共有したくなる体験を提供

## 顧客価値と販路拡大戦略

### ターゲット層
- ローグライク・パズルゲーム愛好家、ハクスラ好き、独特の世界観を好む層
- コアニーズ：思考力・達成感・心理的没入感

### ユニークな体験
- **ダーティネスシステム**：UI濁り・グリッチ・極限状態で視覚的変化を演出
- **責任者Socksとピンチ体験**：戦略上重要なSocksがピンチに陥るドラマ
- **希少イベント**：レア靴下契約や特殊寸劇、心理的示唆を付与
- **自然に語りたくなる瞬間**：SNSや動画で共有されやすい演出をパターン化して挿入

### 長期的エンゲージメント
- **ステータス持越し**による継続的な成長感の提供
- **持久戦ベースの進行設計**
  - 成長曲線は「指数関数的」ではなく「対数関数的」に設計
  - プレイ初日：到達可能現場数 *約3*
  - 継続30日：到達可能現場数 *約6*
  - 継続1年：到達可能現場数 *約12*
- **リプレイ性と語りどころの維持**
  - レアイベントのランダム発生

## ゲーム目的
- 4枚の Socks のうち **責任者Socksが脱出ハッチに到達** で現場クリア
- すべての Socks が力尽きるとゲームオーバー
- 各現場には **PARスコア** を設定し、実手数との差で評価
- 手数上限なし、クリア後は次の現場へ進行

---

## プレイヤーアクション
1. **移動操作**
   - 任意の Socks を選び隣接ノードへスライド
   - エッジ上で靴下同士がすれ違えない単線制約
   - START位置4箇所、責任者Socksはランダム決定

2. **イベント発生**
   - ノード到達時に自動で発生
   - 演出は心理的示唆・希少報酬予兆として機能
   - イベント中の Socks は操作不可

3. **並列性**
   - 他の Socks がイベント中でも残りは操作可能
   - 全員イベント中でのみ待機状態

## ノードの種類

| 種類         | 説明                                                 |
| ------------ | ---------------------------------------------------- |
| START        | プレイヤー投入地点（4箇所）                          |
| HATCH        | 脱出地点。偽装アイコンで表示                         |
| ENEMY        | 自動戦闘。勝敗はステータス比較で決定                 |
| DIRT_POOL    | 汚れ溜まり。ダーティネス上昇・キャラ強化、UI変化あり |
| PURIFICATION | 浄化罠。ダーティネス減少・キャラ弱化、UIクリア化     |
| WARP         | ランダム転移。戦略計画に影響                         |
| SKIT         | 状態依存寸劇。希少報酬や心理的示唆を付与             |

## ダーティネスシステム
- **段階的UI変化**：軽度→中度→重度→極限
- **ゲームプレイへの影響**：移動速度低下、ヒント消失、ノード情報マスク
- **心理的ワクワク**：極限状態や変化の視覚化が自然な語りどころに

---

## アツ演出と心理的示唆
- パチスロインスパイア
- **例**:
  | テンプレート     | 目的           | モジュール例                           |
  | ---------------- | -------------- | -------------------------------------- |
  | HATCH接近        | 到達示唆       | 色変化ログ、軽振動、控えめ音           |
  | 希少イベント予兆 | 発見のワクワク | ログリールの逆回転、端グリッチ、環境音       |

- 発生条件は HATCH距離、ダーティネス、希少イベント残数などに応じ調整
- モジュール内でランダム選択を残し「予測可能だが完全には読めない」体験

---

## 希少イベント・報酬設計
- レア靴下契約、特殊寸劇、心理的示唆など多様
- 発生頻度低く、発見の喜びを重視
- 演出はシンプルながらインパクト大（視覚シンボル＋サウンド＋短テキスト）
- 調整可能な確率パラメータで、ユーザー反応に応じて最適化可能

---

## UI/UXガイドライン
| 課題               | 解決策                                             |
| ------------------ | -------------------------------------------------- |
| 情報過多           | ネットワークグラフ＋ログで整理。詳細は到達時に開示 |
| 行動の影響が不明瞭 | イベント結果や環境変化をログで即時表示             |
| 戦略学習が難しい   | 行動履歴を記録、振り返り可能                       |
| 可視性の低下       | 危険やダーティネス変化を色・フォントで強調         |
| 理不尽感回避       | 確率要素は希少イベントのみ、発生確率は明示         |

---

## 拡散・プロモーション戦略
- **自然発生型の語りたくなる瞬間**を演出で提供
- **SNS・実況映え要素**：ダーティネス極限、希少イベント、責任者ピンチ、ハッチ接近テンプレート
- **最小リソース運営**：演出はテンプレート化＋モジュール化、希少イベント確率はパラメータ化
- **コンテスト型拡散**：不定期（年3~4回）で実況・プレイ動画コンテストを開催

---

## 運営方針
- **新GimmickやSkit追加**：3ヶ月ごとに少量追加、少ない工数で長期リプレイ性を維持
- **ユーザーフィードバック**：App Storeレビュー、SNS言及を観察、重大バグ優先対応、機能追加は計画的に

## 裏コマンド・隠し示唆システム

Dirty Socks には、特定状況下でのみ作動する「隠し示唆」が存在します。  
これは **パチスロの隠しタッチ示唆** に着想を得た、SNS映えと語りどころを強化する仕掛けです。  
示唆は「確定」ではなく「ほのめかし」に留め、プレイヤー体験に余白を残します。

### トリガー例
- **特定状況＋画面タップ**
  - 責任者Socksが瀕死状態、かつダーティネス「重度」
  - 画面端を *3回タップ* すると、HATCH方向が一瞬点滅
  - 1プレイ1回まで

- **デバイスシェイク（iPhoneを振る）**
  - 全員がイベント中で操作不能なタイミング
  - 振動で「遠くから風の音」ログ表示、HATCH方向に応じて端末バイブが短く震える

- **スワイプ隠し入力**
  - START地点に戻った際、特定の順序で上下左右スワイプ
  - 「靴下の古い記憶」演出が流れ、探索ログに追加ヒントを付与

- **無操作時間トリガー**
  - 一定時間（例：15秒）放置するとUIがにじむ
  - にじんだ方向がHATCHのおおよその位置を示唆

### 演出設計ガイドライン
- **非公開性**：ゲーム内で説明せず、ユーザー発見に委ねる
- **効果の軽さ**：確定情報ではなく「方向感・示唆」のみ
- **SNS拡散力**：「偶然見つけた裏コマンド」をシェアしたくなる瞬間を創出
