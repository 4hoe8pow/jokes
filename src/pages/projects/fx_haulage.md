## 3. 想定ユースケース
1.  Houdiniで作成した爆発・煙・魔法・水しぶき等のエフェクトをAEに効率的に持ち込む
2.  Null / Locator / カメラ情報をAEで正確に再現（座標ズレゼロ）
3.  Houdiniレンダーパス（OpenEXRマルチパス）をAEに自動整理し、素材管理を効率化
4.  差分更新対応で、Houdiniでの再レンダリング後もAE上で自動更新
5.  非エンジニアでも直感操作可能なシンプルUIで、学習コストを最小化

---

## 4. 現行Houdini_2_AEカバー範囲（参考）
-   **入力形式**
    -   カメラ / Null（Locator）の位置・回転・スケール
-   **出力形式**
    -   JSON（AE用スクリプト）
    -   EXRマルチパス（互換性・自動整理は限定的）
-   **課題**
    -   Houdini Z-Up → AE Y-Up の座標ズレ発生
    -   スケール補正が手動で、ヒューマンエラーの要因となる
    -   最新Houdini / AE バージョン非対応
    -   UIなし、操作が非直感的で学習コストが高い
    -   Alembic / FBXなど他形式非対応で連携の幅が狭い

## 5. 新プラグイン仕様

### 5.1 入力形式（Houdini側の出力）

| データタイプ                  | 形式             | 補足                                                                                                                                                                                                                                                                                                                                                                                              |
| :---------------------------- | :--------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Null / Locator                | JSON             | 位置・回転・スケール・親子階層情報を保持。Houdini側でベイクされたキーフレームの補間タイプ（線形/ベジェ）も含む。                                                                                                                                                                                                                                                                                  |
| カメラ                        | JSON             | 位置・回転・スケールに加え、焦点距離・絞り・レンズシフトを自動補正。Houdini側でベイクされたキーフレームの補間タイプも含む。                                                                                                                                                                                                                                                                       |
| パーティクル / 簡易ジオメトリ | Alembic (.abc)   | Houdiniで生成されたエフェクト系ジオメトリ（爆発・煙・火花・液体・魔法のエフェクト形状、変形アニメーションを伴う平面やボリュームメッシュなど）。フレーム単位ベイク可能、軽量化オプション付き。キャラクター・プロップ等の詳細メッシュは対象外。AE内での軽量プレビューやレイヤー管理に適用。将来的にはElement 3D等のサードパーティ製3Dプラグイン連携も視野に入れるが、現時点ではプラグイン依存なし。 |
| レンダーパス                  | OpenEXR 16/32bit | HoudiniレンダーパスをOpenEXR形式で出力。マルチパス情報（パス名、レンダーパス識別子など）を保持し、AEでの自動整理に対応。カメラ情報（焦点距離、位置）やフレーム番号等のメタデータも含む。                                                                                                                                                                                                          |
| 将来対応候補                  | FBX              | ジオメトリ・カメラ情報の拡張互換用。基本はカメラ/Null情報のみ抽出してJSON的に処理し、3Dジオメトリは外部プラグイン（Element 3D, Cinewareなど）との連携を想定。Alembic中心の軽量運用を基本としつつ、将来的な拡張性を確保。                                                                                                                                                                          |

### 5.2 AE側動作
-   **座標・スケール変換:**
    -   Null / カメラの座標・回転・スケールをHoudiniの情報を忠実に再現。
    -   Houdini Z-Up → AE Y-Up の座標系自動変換。
    -   Houdini単位（m） → AE単位（px相当）の自動補正。
-   **レンダーパス管理:**
    -   OpenEXRマルチパスを自動でフォルダ／コンポジションに整理。EXRメタデータ（パス名）を基に、コンポジション名・レイヤー名にパス名を自動付与。
    -   AEプロジェクトへの自動配置、名前空間管理。
-   **差分更新:**
    -   Houdiniで再出力された際にAE内の既存オブジェクトやレンダーパスを自動更新。
    -   座標・スケール・キーフレームも再適用。
-   **キーフレーム適用:**
    -   JSONにベイクされたキーフレーム情報をAEに適用。
    -   補間タイプを「線形」または「ベジェ」から選択可能。これによりHoudini側のカーブ形状に近いスムーズなアニメーションを再現。将来的にはHoudini側で埋め込まれた補間情報を忠実に再現する機能も検討。
-   **メタデータ活用:**
    -   EXRメタデータに含まれるカメラ情報を参照し、AE内カメラの自動生成やマッチムーブに活用。
    -   フレーム番号を利用した差分更新時の読み込みチェックやエラー特定に活用。

### 5.3 UI
-   **形式**：CEPパネル（HTML/JS）
-   **操作ボタン**
    -   JSON読み込み
    -   Alembic読み込み
    -   OpenEXR読み込み
    -   プレビューON/OFF（軽量表示モード切り替え）
    -   差分更新ON/OFF
    -   座標系・スケール自動補正ON/OFF
    -   キーフレーム補間タイプ選択（線形/ベジェ）
-   **状態表示**
    -   インポート成功／失敗（ファイルごとのステータス）
    -   読み込んだNull／カメラ／EXR数
    -   **エラー／警告ログ:**
        -   どのファイルでエラーが発生したか（JSON/Alembic/EXR）。
        -   どのオブジェクト／レイヤーに問題があるか（Null名、カメラ名、EXRパス）。
        -   どの値／属性が不正か（座標、回転、スケールの異常値、欠損パスなど）を具体的に表示。
        -   ユーザーが問題箇所を特定し、手動での原因究明の手間を省くことで作業効率を向上。ログは保存してチームで共有可能。

### 5.4 差別化ポイント
1.  **Null / カメラ座標ズレゼロ保証:** 現場での手動調整工数を完全に排除。
2.  **OpenEXRマルチパスの自動整理と高精度なメタデータ活用:** 素材管理の煩雑さを解消し、後工程の作業効率を大幅に向上。
3.  **非エンジニアでも操作可能な直感UI:** 学習コストを低減し、幅広いユーザーが導入可能。
4.  **差分更新対応で再レンダリングも簡単:** 試行錯誤のプロセスを効率化。
5.  **Houdini最新バージョン対応:** Houdiniの進化に合わせて継続的にサポート。
6.  **AE CC2020〜最新対応:** 幅広いAfter Effects環境で利用可能。
7.  **JSON + EXRによる軽量・高速処理:** 複雑なシーンでもスムーズな連携を実現。
8.  **詳細なエラーログと補正機能:** 問題発生時の原因究明を迅速化し、ワークフローの安定化に寄与。

## 6. 開発アーキテクチャ

### Houdini側

| コンポーネント                 | 処理内容・役割                                                                                                                                                      |
| :----------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| JSON Export (Null/Camera)      | NullやLocator、カメラの位置・回転・スケール・親子階層情報、およびベイクされたキーフレームの補間タイプをJSON形式で出力。AEの座標系に合わせた変換をオプションで実行。 |
| Alembic Export (Geo/Particles) | パーティクルや簡易ジオメトリをAlembic形式でエクスポート。フレーム単位でベイク可能、軽量化オプションあり。                                                           |
| OpenEXR Export (マルチパス)    | HoudiniレンダーパスをOpenEXR形式（16bit/32bit）で出力。マルチパス情報、カメラ情報、フレーム番号等のメタデータを保持し、AEでの自動整理に対応。                       |

### AEプラグイン側（C++ / CEP）

| コンポーネント       | 処理内容・役割                                                                                                                                                                                                   |
| :------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| JSON Parser          | Houdiniから出力されたJSONを読み込み、AE内でNull / カメラオブジェクトを生成。座標系（Z-Up→Y-Up）・スケール補正を自動適用。キーフレーム情報をAEのキーフレームとして線形／ベジェ補間を選択し適用。                  |
| Alembic Loader       | Alembicファイルを読み込み、AE 3Dレイヤーとして生成。フレームベイクされたジオメトリ・パーティクルを再現し、AE上で軽量表示。                                                                                       |
| EXR Loader           | OpenEXRマルチパスを読み込み、AEのフォルダやコンポジションに自動整理。レンダーパスごとのレイヤー管理もサポート。EXRメタデータ（パス名、カメラ情報、フレーム番号）を解析し、AEオブジェクトの生成や情報付与に活用。 |
| 差分更新 / 自動補正  | Houdini再出力時にAE内の既存オブジェクトやレンダーパスを自動更新。座標・スケール・キーフレームも再適用。                                                                                                          |
| UI Panel (CEPパネル) | JSON/Alembic/EXR読み込み、差分更新ON/OFF、プレビューON/OFF、キーフレーム補間タイプ選択などの操作ボタンを提供。読み込み状況・エラー・警告の具体的ログ表示も行う。                                                 |

### 処理フローの概要

1.  Houdiniでエフェクトやカメラ・Nullを作成。
2.  JSON / Alembic / OpenEXR形式でエクスポート。
3.  AEプラグインでファイル読み込み。
    -   JSON → Null / カメラ生成（座標・スケール・キーフレーム補正適用）。
    -   Alembic → 3Dレイヤー生成（軽量表示）。
    -   EXR → 自動コンポジション・フォルダ配置（メタデータ活用）。
4.  差分更新が有効なら、Houdiniでの再出力時にAEプロジェクトを自動更新。
5.  CEPパネルで操作・状態確認、エラーログの参照。
