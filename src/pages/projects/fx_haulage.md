## 3. 想定ユースケース
1. Houdiniで作成した爆発・煙・魔法・水しぶき等のエフェクトをAEに効率的に持ち込む
2. Null / Locator / カメラ情報をAEで正確に再現（座標ズレゼロ保証）
3. Houdiniレンダーパス（OpenEXRマルチパス）をAEに自動整理、素材管理を効率化
4. 差分更新対応で、Houdiniでの再レンダリング後もAE上で自動更新
5. 非エンジニアでも直感操作可能なシンプルUIで、学習コストを最小化
6. 可逆性（Houdini → AE → Houdini）のテストに耐えうる堅牢性確保

---

## 4. 現行Houdini_2_AEカバー範囲（参考）
- **入力形式**
    - カメラ / Null（Locator）の位置・回転・スケール
- **出力形式**
    - JSON（AE用スクリプト）
    - EXRマルチパス（互換性・自動整理は限定的）
- **課題**
    - Houdini Z-Up → AE Y-Up の座標ズレ
    - スケール補正が手動で、ヒューマンエラー発生
    - 最新Houdini / AE バージョン非対応
    - UIなし、操作が非直感的
    - Alembic / FBXなど他形式非対応で連携幅が狭い

## 5. 新プラグイン仕様

### 5.1 入力形式（Houdini側の出力）

| データタイプ                  | 形式             | 補足 |
| :---------------------------- | :--------------- | :--- |
| Null / Locator                | JSON             | 位置・回転・スケール・親子階層情報、キーフレーム補間タイプ（線形/ベジェ）、UID / チェックサム付き。差分更新に必須。 |
| カメラ                        | JSON             | 位置・回転・スケール、焦点距離・絞り・レンズシフト自動補正。キーフレーム補間、UID/チェックサム付き。 |
| パーティクル / 簡易ジオメトリ | Alembic (.abc)   | エフェクト系ジオメトリ（平面/ボリューム/パーティクル）フレーム単位ベイク、軽量化オプション。将来的にElement 3D連携を想定。 |
| レンダーパス                  | OpenEXR 16/32bit | マルチパス情報、カメラ情報、フレーム番号等メタデータ保持。AEで自動整理・差分更新対応。 |
| 将来対応候補                  | FBX              | カメラ/Null情報抽出、拡張互換用。3Dジオメトリは外部プラグイン連携を想定。 |

#### 5.1.1 JSON UID / Checksum設計
- `uid`: 永続ID（UUID v4 または安定ハッシュ）  
- `checksum`: bakedデータ + 行列 + メタ情報のSHA256  
- マッチング順序：
  1. uid 一致 → マッチ確定
  2. checksum 一致 → 再出力オブジェクトとしてマッチ
  3. name + parent_uid + approx transform → フォールバック
  4. どれも一致せず → 新規インポート

### 5.2 AE側動作

#### 5.2.1 座標・スケール・回転変換
- **4x4 行列ベース変換**を主体に処理し、Eulerの不整合やgimbal問題を回避
- Houdini Z-Up → AE Y-Up 自動変換
- Houdini 単位（m） → AE px相当自動補正
- 行列変換のメタを保持（source_up_axis / target_up_axis / source_units / target_units）

#### 5.2.2 差分更新
- UID/Checksumをベースに既存オブジェクトとマッチング
- 許容閾値（tolerance）超の差分のみ更新
    - translation tolerance = 1e-4 m
    - rotation tolerance = 1e-3 deg
    - scale tolerance = 1e-5 relative
- ユーザー編集フラグ保護モードを提供
- 差分適用前にプレビュー表示

#### 5.2.3 キーフレーム適用
- 補間タイプ（linear / bezier / step）を忠実に適用
- AEの自動最適化は行わず、Houdiniカーブを忠実に再現

#### 5.2.4 レンダーパス管理
- OpenEXRマルチパスをコンポジション・フォルダに自動整理
- EXRメタデータでAEカメラ生成・マッチムーブ支援
- 差分更新時にフレーム番号とUIDで一致判定

### 5.3 UI
- CEPパネル（HTML/JS）
- ボタン：
    - JSON / Alembic / OpenEXR読み込み
    - プレビューON/OFF、差分更新ON/OFF
    - キーフレーム補間タイプ選択
- 状態表示：
    - インポート成功／失敗
    - 読み込んだNull / カメラ / EXR数
    - エラー／警告ログ（オブジェクト単位、属性単位）
- 差分プレビュー表示（追加/削除/変更を色分け）
- 差分サンドボックスモード、戻せる差分スナップショット

### 5.4 差別化ポイント
1. Null / カメラ座標ズレゼロ保証（行列ベース＋公差管理）
2. UID/Checksumによる差分更新で“微妙に違う”アンチパターンを回避
3. OpenEXRマルチパス自動整理＋メタデータ活用
4. 非エンジニア向け直感UI、差分プレビュー、保護モード
5. Houdini最新バージョン対応
6. AE CC2020〜最新対応
7. JSON + EXR軽量処理で大規模シーンも高速
8. 詳細ログ＋差分履歴で原因究明容易
